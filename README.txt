При каком размере матрицы n на n вычисления затруднительны?

1) n=10; Время = 0 ms.
2) n=100; Время = 0 ms.
3) n=1_000; Время = 1 ms.
4) n=10_000; Время = 137~142 ms.
4.1) n=5_000; Время = 35 ms.
4.2) n=7_500; Время = 81 ms.
5) n=100_000 - Переполнение! Матрица не была сгенерирована
5.1) n=50_000 - Переполнение! Матрица не была сгенерирована
5.2) n=25_000 - Переполнение! Матрица не была сгенерирована
5.3) n=15_000; Время = 303~311 ms.
5.4) n=20_000; Время = 541~567 ms.

Вывод: Значительные затруднения начинаются со значение n>=10_000;
 значение от n>20_000 вызывают переполнение памяти
